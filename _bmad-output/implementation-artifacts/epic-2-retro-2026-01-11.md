## Challenges & Lessons Learned

### Challenge 1: Firestore Real-time Sync Complexity

**Issue:** Managing onSnapshot listeners with debounced saves created timing issues in early iterations.

**Resolution:** Implemented offline queue pattern with localStorage fallback for failed Firestore writes.

**Lesson:** Always plan for offline-first when dealing with real-time sync. Users expect their changes to persist even with connectivity issues.

### Challenge 2: Role Detection Edge Cases

**Issue:** Users with multiple roles or undefined roles initially broke dashboard rendering.

**Resolution:** Created `getDashboardRole()` utility that gracefully falls back to 'user' role.

**Lesson:** Always have sensible defaults for role-based features. Users should never see a broken UI due to role configuration issues.

### Challenge 3: Widget Registry Growth

**Issue:** As widgets grew from 18 to 26, maintaining the registry became more complex.

**Resolution:** Established clear widget registration pattern with required fields (id, component, titleKey, defaultColSpan).

**Lesson:** Document patterns early. The WidgetRegistry pattern should be documented for future widget additions.
